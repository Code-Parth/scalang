# @scalang/template

Next.js template app for serving multilingual [Scalar](https://scalar.com) API references with [Scalang](https://github.com/Code-Parth/scalang).

This is the template scaffolded by `scalang create`. It provides dynamic locale routing, Scalar theme configuration, and the `@scalang/react` language selector out of the box.

## Getting Started

### 1. Generate Specs

Before running the app, generate translated specs from your OpenAPI source:

```bash
bunx @scalang/cli generate
```

This creates translated spec files in `public/specs/` and a `manifest.json`.

### 2. Start Development Server

```bash
bun dev
```

Visit `http://localhost:3000/en` (or any configured locale).

## How It Works

- **Dynamic routing** вЂ” `[lang]/page.tsx` reads the locale from the URL, loads the corresponding spec from `public/specs/{lang}.json`, and renders it with `ScalangApiReference`.
- **Manifest-driven** вЂ” Available locales come from `public/specs/manifest.json`, generated by the CLI.
- **Scalar config** вЂ” Theme, layout, dark mode, and other Scalar options are read from `.scalang-config` and passed through to the Scalar viewer.
- **Metadata** вЂ” Page title and description are derived from each spec's `info.title`.

## Configuration

All configuration lives in `.scalang-config` at the project root. See [`@scalang/schema`](https://www.npmjs.com/package/@scalang/schema) for the full schema.

Example:

```json
{
  "spec": "./petstore.json",
  "sourceLocale": "en",
  "locales": ["fr", "de", "ja"],
  "outputDir": "./public/specs",
  "scalar": {
    "theme": "purple",
    "layout": "modern",
    "darkMode": true
  }
}
```
